react Context

发表于 2019-03-10 分类于 web ， react

作用

Context的作用是共享组件树的全局数据，如主题、语言等。Context使我们无需为了在组件间传递数据，而为每层组件手动添加props。。

API

创建

React.createContext

  • 用来创建Context对象，以供其他组件订阅。

  • 订阅Context对象的组件被称作消费组件。

  • 一个Context可以被多个消费组件订阅

更新

Context.Provider

  • Context.Provider组件用来更新Context的值。

  • 当Provider的值变化时，其内部的所有消费组件会被重新渲染。

  • Provider及其内部消费组件不受制于shouldComponentUpdate函数，因此当 consumer 组件在其祖先组件退出更新的情况下也能更新。

订阅

Context.Consumer

当组件为函数组件，使用此api订阅Context对象。

Class.contextType

  • 通过给Class.contextType赋值来订阅单一的Context对象，进而可以在任何生命周期中，使用this.context来获取订阅的Context对象的值。

  • 消费组件会使用离自己最近的对应Context.Provider的value值。

Context.displayName

context对象的displayName属性，React DevTools使用该属性来确定context要显示的内容。

在消费组件中更新Context

可以通过 context 传递一个函数，使得 consumers 组件通过此函数更新 context。

订阅多个Context

React需要每个context对象时一个单独的节点，以确保context快速渲染。

注意

Context会影响性能，不要滥用。

为了避免不必要的重复渲染，应使用state来配置Provider组件的value属性。
