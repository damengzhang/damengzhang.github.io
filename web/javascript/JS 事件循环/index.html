JS 事件循环

发表于 2020-08-20 分类于 web ， js

1 事件循环

事件循环是 JS 主线程处理异步任务的过程，包含如下元素：

  • 堆
  • 调用栈（也叫执行栈）
  • 消息队列（也叫任务队列、宏任务队列、callback queue）
  • 微队列

3 调用栈

函数被执行时，会被压入调用栈 执行。

上面代码执行过程：

  1. 当调用 bar 函数时，bar 被压入栈中。
  2. 当 bar 调用 foo 时，foo 被压入栈中。
  3. 当 foo 执行完成时，foo 被弹出栈。
  4. 当 bar 执行完成时，bar 也被弹出栈，此时栈被清空了。

4 消息队列

消息队列（也叫任务队列、宏任务队列、callback queue）由待处理的回调函数组成，事件循环会依次处理消息队列中的回调。

当回调被处理时，会被移出消息队列，然后压入调用栈中执行。

4.1 宏任务

宏任务（MacroTask，也叫 Task），由下面类型的代码形成，并被加入到宏任务队列：

  • 主代码块，比如从一个控制台，或在 script 标签中运行代码

  • 被触发的事件回调

  • setTimeout / setInterval / setImmediate

5 微任务队列

微任务队列由微任务组成，事件循环会依次处理微任务队列中的微任务。

5.1 微任务

微任务（MicroTask，也叫 Jobs），下面类型的代码属于微任务，并被加入微任务队列：

  • Promise

  • process.nextTick

  • Object.observe

  • MutationObserver

  • queueMicrotask

    • 使用场景

      • 代码执行结果的一致性

      • 串行执行任务

6 执行顺序

JS 主线程执行过程：

  1. 同步脚本

  2. 执行微任务队列，直到微任务队列为空

  3. 当微任务队列中的任务全部执行完成后，从消息队列中取下一个任务开始执行

  4. 当前任务完成后，执行浏览器渲染

  5. 浏览器渲染完成后，继续步骤 2

当前 task > microtask（直到队列为空才进行下一步） > 渲染绘制 -> 下个 task
